(win版)hexo博客搭建及GitHub远端部署

Created2022-06-29|Updated2022-07-04|Hexo Blog
|Post View:

学习搭建博客文章

内容：记录本人自己搭建hexo博客的一个过程（Windows系统里的搭建）

下载安装Node.js

Node.js的官方Node.js (nodejs.org)点击下方图片中红色的框进行下载。（16.15.1这个版本下载下来有点问题，下载一个以前的版本）

点击下方图片的红色框。

在下方红色框中搜索我们要的版本，这里我选择的版本为16.14.2，然后点击下载。

点击下方图片红色框进行下载，下载完之后进行安装即可，安装过程一直 next 即可，中间有协议之类的选择我同意。

安装完成之后，WIN+R键输入 cmd 打开命令窗口，输入 node -v 查看node的版本号，如下方图所示出现你所安装的版本号说明你已经安装成功。

然后在命令行中输入 npm -v 查看 npm 包的版本号，如果正确输出版本号，说明你已经可以开始下一步了。

接下来就是用 npm 包管理器来安装 Hexo 博客框架，因为国内的访问速度比较慢，所以这里要利用 npm 安装一个 cnpm（设置淘宝镜像源，访问速度比较快）

在命令行中输入 npm install -g cnpm –registry=https://registry.npm.taobao.org

等待安装即可，安装完成之后可以输入 cnpm -v 查看 cnpm 的版本号，和前面的一样，正常显示版本号的话表示安装完成。（下方图片红色框的内容是我已经下载过了，所以下载显示和下面不一样）

安装Git

git 下载地址：Git - Downloads (git-scm.com)

打开网站后点击下方红色框的内容，进行下载。（点击后会跳转另一个网页，根据需要选择相应的版本）

下载好后，打开程序进行安装，安装过程一直点 next，安装位置自己选好，只有图片下方的内容需要勾选，其余点 next 即可。

安装 Hexo博客框架

Win+R 键输入 cmd 打开命令窗口，在命令行输入 cnpm install -g hexo-cli 等待安装，安装结果如下:

可以用 hexo -v 命令来验证 Hexo 石头安装成功

初始化博客文件夹

找个地方创建一个文件夹作为博客的文件夹，例如下方图片里。

然后在文件夹上的导航栏，输入cmd，回车进入命令行。

在命令行输入 hexo init 初始化博客框架。

在本地 Hexo 运行博客

在命令行中输入 hexo s 回车（复制这个网址到浏览器）

我们的个人博客已经在我们本地服务器的 4000 端口上i开始运行

创建一篇新文章

博客已经搭建完毕，接下来就可以创建文章，写文章。

创建一篇新文章只需要在命令行中输入：hexo n “文章的名字” （这里的n是new的缩写，当然你也可以写全）

上方图片的紫色字体是存放位置，我们按着路径找到文件。(这里我用vs打开的)

文件编辑好记得保存，之后我们回到命令行，输入 hexo 经典三语句。(本地部署)

然后像之前一直，打开浏览器，输入网址，可以看到我们刚刚写好的文章。

注册登录 GitHub 账号

在网站里搜索 Github，出现下面的页面，红色的框是登录，蓝色的框是注册（打开GitHub会经常失败，这里有两个简单的办法1、下载uu加速器里搜索 “学术” 开始加速，2、下载steam++里有Github加速）

有账号的直接登录就行，没有的进行注册如图输入自己的邮箱即可。

创建一个 Github 仓库

点击 Github 页面右上角的+号，在出现的菜单中选择 Newrepository

然后出现的页面中，填写创建仓库的信息，一步一步来。

1、首先是仓库的名字，因为是部署个人博客的，所以 Github 仓库的名字有一定的命名要求，这个仓库的名字必须是你的用户名.github.io，比如我的用户名是 ShuYePL ，那么我的仓库名就必须是 Minaku01.github.io
2、然后是仓库的介绍，这个看你自己心情了，随便写都可以
3、选择是否将仓库公开，这个，你既然是做个人博客，当然是要公开给别人看的啦，所以要选 public ，这个默认选项。

以上完成后其他选项就不要动了，直接选择页面最下面的绿色按钮 Create repository 。

安装一个 Git 的部署插件

回到命令行，输入 cnpm install hexo-deployer-git –save 等待安装完成

设置博客的配置文件_config.yml

打开个人博客文件夹下的 _config.yml 文件（用记事本打开就行），拉到文件的最下方，找到 # Deployment，在这个地方进行一些配置

1、首先的 type 选项，在它的后面填上 git
2、然后在 type 选项的下一行，写上一个属性 repo: ，后面空一行，填上你仓库的地址，仓库的地址返回你刚刚创建好的仓库页面，在下图红色方框内的内个就是你的仓库地址了，直接粘贴，粘贴到 repo 属性后面。

3、再在配置文件的下一行中写上 branch: master
记得保存！！！！！！！！！！！！！

git 配置用户名和邮箱

如果没有这一步的话，下面的远端部署就会失败

在 Windows 端直接右键选择 Git Bash，或者如图直接搜索

在出现的界面中依次输入以下的两条指令

git config –global user.name “你的 Github 用户名”
git config –global user.email “你注册 Github 账户的邮箱”

如下：

远端部署

hexo 三件套（远端部署）

hexo clean
hexo g
hexo d
这里实际也可以直接 hexo d

第一次运行 hexo d 命令的话会要求你登录 Github 账户，这里有两种登录方式，我选择了第二种

点击之后会出现一个验证码，我们先点击验证码下方的内个连接，会跳出一个浏览器窗口，在浏览器窗口中，我们点击绿色的按钮

在弹出来的窗口中输入内一串验证码，然后出现了一个输入密码的页面

输入你 Github 账户的密码

上传的任务会继续执行，下方图片红色框粘贴到浏览器，出现和本地部署一样就成功了。
